# B2B2C商城项目军规
> 所有项目参与人员必须严格遵守以下规定，违者必究

## 一、核心原则
1. **用户操作最小化**：系统操作人员仅需执行关键性审核类、决策类操作，所有重复性、机械性的导入/部署/配置等操作由自动化流程完成，禁止要求操作人员手动执行导数据、改配置等重复劳动
2. **问题闭环原则**：所有问题必须有明确的责任人、处理时限、验证标准，处理完成后必须提交验证结果，不得出现悬而未决的问题
3. **安全第一原则**：所有代码提交必须经过安全扫描，敏感数据不得出现在代码、配置文件、日志中
4. **质量保障原则**：所有新功能必须包含单元测试、集成测试，测试覆盖率不低于80%

## 二、开发规范
1. 代码必须符合项目统一编码规范，提交前必须经过格式化、静态检查
2. 数据库操作必须使用参数化查询，禁止SQL拼接，防止SQL注入
3. 前端请求必须包含参数校验，后端必须二次校验，禁止信任前端提交的任何数据
4. 所有接口必须返回统一的响应格式，错误信息必须友好，不得泄露系统内部细节

## 三、部署规范
1. 所有环境部署必须使用自动化脚本，禁止手动修改服务器配置
2. 数据库变更必须使用SQL脚本，禁止直接在线上库执行DDL/DML操作
3. 上线必须经过测试环境验证、预发环境验证，才能正式上线
4. 上线必须有回滚方案，出现问题5分钟内必须能够回滚到上一个版本

## 四、运维规范
1. 所有线上操作必须有日志记录，操作前必须备份，操作后必须验证
2. 敏感操作必须双人复核，禁止单人执行数据库删除、配置修改等高危操作
3. 系统监控必须覆盖CPU、内存、磁盘、网络、接口响应、错误率等核心指标
4. 故障处理必须遵循"先恢复，后排查"原则，优先保障系统可用，再排查根因

## 五、数据规范
1. 所有数据导入导出必须由自动化流程完成，禁止人工手动操作生产库
2. 测试数据必须脱敏，不得使用真实的用户数据、支付数据等敏感信息
3. 数据库备份必须每天自动执行，备份文件必须异地存储，定期验证可恢复性
4. 数据删除必须遵循软删除原则，重要数据禁止物理删除，必须保留操作日志
