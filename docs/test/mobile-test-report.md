## 6. 代码质量问题

### 6.1 Vue Prop类型警告 ✅
**问题**: src/components/ProductCard.vue
```
首页推荐区域错误地将product.maxPrice作为originalPrice传入，
语义上maxPrice代表价格区间上限而非原价，应使用product.originalPrice
```
**修复方案**: 将首页ProductCard的:original-price绑定改为使用`product.originalPrice`

### 6.2 bankcard.ts默认导入错误 ✅
**问题**: src/api/bankcard.ts
```
使用import http from '@/utils/request'（默认导入），
但request.ts中http是命名导出
```
**修复方案**: 改为`import { http } from '@/utils/request'`

### 6.3 getCommentList调用签名不匹配 ✅
**问题**: src/pages/product/index.vue
```
getCommentList(spuId, { page: 1, limit: 3 }) 使用两个参数调用，
但API定义只接受单个对象参数 { spuId, page, limit }
```
**修复方案**: 改为`getCommentList({ spuId, page: 1, limit: 3 })`

### 7.4 401 Unauthorized分析

| 接口 | 是否需要登录 | 401是否合理 | 处理方式 |
|------|-------------|------------|----------|
| `/api/v1/member/message/unread-count` | 是 | ✅ 合理 | 前端使用Promise.allSettled兜底，未登录时隐藏红点 |
| `/api/v1/member/auth/register` | 否 | ❅ 不会触发401 | SecurityConfig中`/api/v1/member/auth/**`已配置permitAll |
| `/api/v1/member/auth/sms/send` | 否 | ❅ 不会触发401 | 同上，属于auth路径 |

**结论**: 401报错主要发生在未登录用户访问需认证接口（消息、购物车等），属正常安全行为。前端已增加容错处理。

### 7.5 短信验证码安全机制

| 安全措施 | 实现方式 |
|---------|---------|
| 发送频率限制 | Redis 60秒间隔（SMS_INTERVAL key） |
| 每日发送上限 | Redis计数器，每手机号每日10次 |
| 验证码过期 | 数据库expire_at字段 + Redis 5分钟TTL |
| 验证尝试次数 | 数据库verify_attempts字段，最多5次 |
| 一次性使用 | 验证成功后标记verified=1，删除Redis缓存 |
| 审计追踪 | 所有验证码记录持久化到sms_code表 |
| IP记录 | 记录请求IP到数据库 |
