# 第三期：特色功能

> 版本：v1.0 | 更新日期：2026-04-30 | 预计周期：5-6周 | 前置依赖：第二期完成

## 一、本期目标

实现功能列表中的差异化特色功能，提升商城竞争力，包括拍卖、夺宝奇兵、超值礼包、预售、社区团购、储值卡/充值卡、实体门店、白条分期等模块。

### 1.1 核心原则

- **差异化竞争**：本期功能为商城特色卖点，区别于同质化平台
- **交易安全**：拍卖/夺宝等涉及资金的功能，安全机制必须到位
- **社交裂变**：社区团购自带社交属性，降低获客成本
- **金融合规**：白条/储值卡等金融功能需确保合规

### 1.2 验收标准

- [ ] 拍卖活动可正常竞拍，出价记录实时更新
- [ ] 夺宝奇兵可正常参与，出价规则正确
- [ ] 超值礼包可正常购买，库存不足有提示
- [ ] 预售定金+尾款流程完整
- [ ] 社区团购团长可管理社区、处理订单
- [ ] 储值卡/充值卡可正常发放和使用
- [ ] 门店自提流程完整，提货码验证通过
- [ ] 白条分期可正常使用

## 二、需求分析师（requirements-analyst）工作内容

### 2.1 需求分析重点

1. 拍卖出价规则（起拍价/加价幅度/一口价/保证金）
2. 夺宝奇兵玩法规则（出价范围/唯一最低价判定）
3. 预售定金+尾款流程（定金规则/尾款支付时间/超时处理）
4. 社区团购团长招募、佣金、订单流转的完整规则
5. 白条额度和分期费率计算规则
6. 储值卡/充值卡的使用条件和充值流程
7. 门店自提的提货码生成和验证逻辑

### 2.2 PRD输出要求

输出文件：`docs/prd/PRD-第3期-特色功能.md`

重点包含：
- 拍卖出价与成交规则详解
- 夺宝奇兵算法规则（唯一最低价/取整规则）
- 预售定金+尾款时间轴流程图
- 社区团购完整业务流程（团长→社区→订单→佣金）
- 白条分期计算公式和风控规则
- 门店自提码生成→短信发送→验证完整流程

### 2.3 评审要点

- 拍卖/夺宝的资金安全（保证金退还规则）
- 预售定金是否可退（法律合规）
- 社区团购团长与平台的权责划分
- 白条分期是否涉及金融牌照要求
- 门店自提的库存同步问题

## 四、核心业务流程

### 4.1 拍卖出价流程

```
用户缴纳保证金 → 出价（>= 当前价+加价幅度）
→ 出价记录更新 → 其他用户被超越
→ 拍卖结束 → 最高价者成交
→ 支付尾款（保证金抵扣）→ 发货
→ 未成交者退还保证金
```

### 4.2 预售定金+尾款流程

```
预售活动开始 → 用户支付定金
→ 等待尾款期开始
→ 尾款期内支付尾款（定金抵扣）
→ 商家发货
→ 超时未付尾款：定金不退（可配置）
```

### 4.3 社区团购流程

```
平台创建社区 → 关联团长
→ 团长推广商品给社区居民
→ 居民下单（配送至团长处）
→ 团长收到货物 → 通知居民自提
→ 团长确认交付 → 佣金结算
```

### 4.4 门店自提流程

```
用户下单 → 选择自提门店
→ 支付成功 → 生成提货码
→ 短信发送提货码
→ 用户到店出示提货码
→ 店员扫码验证 → 交付商品
```

## 六、技术要点

### 6.1 拍卖出价并发控制

- Redis Sorted Set 记录出价
- Lua脚本保证出价原子性
- WebSocket实时推送出价更新

### 6.2 夺宝奇兵算法

- 唯一最低价：所有出价中只出现一次的最低价格
- 出价写入Redis Set（去重），活动结束后计算
- 高并发下使用Lua脚本原子判定

### 6.3 预售时间轴控制

- 定金期/尾款期使用Redis Key过期事件驱动状态流转
- 定时任务兜底检查超时未付尾款订单

### 6.4 白条风控

- 额度评估算法（消费记录/信用分/还款历史）
- 逾期自动降额/冻结
- 还款提醒（短信/站内信）
