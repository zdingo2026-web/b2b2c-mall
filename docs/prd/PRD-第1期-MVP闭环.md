## 1. 概述

### 1.1 本期目标

完成B2B2C商城全链路最小可用版本，四端（平台管理端、商户管理端、PC商城、移动端）均能独立运行，实现完整购物闭环。

### 1.2 核心价值

- 验证B2B2C多商户模式的商业可行性
- 建立可扩展的技术架构底座
- 快速上线获取用户反馈

### 1.3 依赖前置条件

- 无（本期为项目首期）

### 1.4 设计规范变更说明 (Calicat v2)

> **TabBar变更**：原方案TabBar为"首页/分类/购物车/我的"，现根据Calicat设计稿调整为"首页/分类/订单/我的"。购物车降为普通页面（从商品详情页底部操作栏或页面内入口进入），订单页提升为TabBar项。设计稿来源：Calicat file_id `2050096196913561600`。

## 4. 业务流程

### 4.1 注册登录流程

```
[PC端]
手机号注册：输入手机号 → 发送验证码 → 输入验证码 → 设置密码 → 注册成功
登录：用户名/手机号/邮箱 + 密码 → JWT Token → 登录成功
微信登录：扫码 → 授权 → 绑定已有账号或新建账号

[移动端]
微信登录：点击微信登录 → 授权 → 获取手机号 → 登录/注册
手机号登录：输入手机号 → 验证码 → 登录/注册
```

### 4.2 下单支付流程

```
选择商品 → 加入购物车
→ 购物车页面选择商品 → 去结算
→ 结算页：选择收货地址 + 配送方式 + 备注
→ 提交订单（按商家拆分为子订单）
→ 选择支付方式（微信/支付宝/余额）
→ 支付成功 → 订单状态变为"待发货"
→ 支付超时（30分钟）→ 订单自动取消 → 库存回补
```

### 4.3 订单状态流转

```
创建订单 → 待付款
  ↓ 支付成功       ↓ 超时/取消
待发货            已取消（库存回补）
  ↓ 商家发货
待收货
  ↓ 确认收货     ↓ 申请退款
已完成           退款中
                   ↓ 退款成功
                  已退款
```

### 4.4 商家入驻流程

```
商家访问入驻页面
→ 填写企业信息（公司名/营业执照/联系人/手机号）
→ 提交申请
→ 平台管理员审核
  → 通过：创建商户账号，短信通知商户
  → 拒绝：短信通知商户原因
→ 商户登录后台 → 完善店铺信息 → 发布商品
```

### 4.5 多商家拆单流程

```
购物车中含A店商品x + B店商品y
→ 结算页展示：
  - A店订单：商品x、运费、小计
  - B店订单：商品y、运费、小计
  - 合计金额
→ 提交后生成：
  - 主订单（全局订单号）
  - 子订单A（A店，独立发货/物流）
  - 子订单B（B店，独立发货/物流）
→ 各子订单独立状态流转
```

## 5. 功能详细说明

### 5.1 用户注册（C-01）

- **输入**：手机号、短信验证码、密码（8-20位，大小写字母+数字）
- **校验**：手机号格式、验证码正确性、密码强度、手机号未注册
- **异常**：验证码错误→提示"验证码错误"；手机号已注册→提示"手机号已注册，请直接登录"
- **安全**：验证码5分钟过期、同一手机号60秒内不可重复发送、密码bcrypt加密

### 5.2 商品详情（C-06）

- **展示**：商品图片相册（多图切换+放大镜）、SPU基本信息、SKU选择器（颜色/尺寸→价格库存联动）、商品描述（PC+移动端详情）、运费计算
- **交互**：选择SKU属性→价格/库存/图片自动切换；加入购物车→弹出成功提示
- **异常**：SKU无库存→显示"暂无库存"不可加入购物车；商品已下架→显示"商品已下架"

### 5.3 购物车（C-07）

- **展示**：按商家分组展示商品、图片/名称/SKU属性/单价/数量小计
- **交互**：全选/单选、数量增减（1-库存上限）、删除、合计金额实时计算
- **异常**：商品下架→标记"已下架"不可结算；库存不足→标记"库存不足"并调整可购数量
- **持久化**：登录用户购物车存数据库；未登录用户购物车存本地（登录后合并）

### 5.4 订单拆单（C-08）

- **规则**：同一结算中的商品按商家分组，每个商家生成一个子订单
- **运费**：每个子订单独立计算运费
- **优惠**：优惠券/积分等在子订单间按比例分摊（第二期）
- **独立操作**：每个子订单独立发货、独立物流、独立退换货

### 5.5 支付（C-09）

- **微信支付**：PC端扫码支付、移动端H5/小程序支付
- **支付宝**：PC端网页支付、移动端H5支付
- **余额支付**：扣减用户余额，余额不足不可选择
- **超时**：30分钟未支付自动取消订单，库存回补
- **回调**：支付网关回调更新订单状态，幂等处理

### 5.6 登录页（A-01 增强）

- **视觉**: 全屏紫蓝渐变背景(#667eea→#764ba2)，白色圆角卡片表单居中
- **表单**: 手机号输入框+"更换"按钮、验证码输入框+"获取验证码"药丸按钮(蓝色60s倒计时)
- **登录按钮**: 蓝靛渐变(#667eea→#764ba2)，圆角
- **协议**: 复选框 + "同意《用户协议》和《隐私政策》"，未勾选时点击登录弹出提示
- **三方登录**: 3个圆形图标(微信绿/QQ蓝/支付宝蓝)，UI占位不接真实三方
- **注册**: 底部"注册账号 >"链接

- **物流卡片**: 蓝色背景(#EFF6FF)，物流公司+单号+状态文字，仅"待收货"状态显示
- **收货地址**: 红色定位图标 + 姓名+电话 + 完整地址 + "默认"蓝色标签(如有)
- **店铺信息**: 店铺头像+名称+"进店逛逛"按钮(蓝色文字+右箭头)
- **商品明细**: 商品图+名称+规格+单价+数量+小计
- **订单信息**:
  - 订单编号 + 一键复制按钮(clipboard API)
  - 下单时间 / 支付时间 / 发货时间(按状态显示)
  - 支付方式(微信/支付宝/银行卡)
  - 发票信息(不开发票/电子发票+抬头)
- **费用明细**: 商品总价 + 运费 + 优惠(红色-¥xx) = 实付金额

  - 缺货: 灰色背景+灰色文字+"缺货"角标，不可点击
- **数量步进器**: 减号(-) + 数字 + 加号(+)，范围1~库存
- **服务说明**: "7天无理由退换" 等文字
- **确认按钮**: 红色(#E11148) "确认加入购物车"，选中完整规格后可点击

## 6. 非功能需求

### 6.1 性能

| 指标 | 要求 |
|------|------|
| 首页加载 | < 2s (PC) / < 1.5s (移动) |
| API响应 | P95 < 500ms |
| 并发 | 支持500 QPS |
| 数据库 | 单表100万级数据量下查询 < 200ms |

### 6.2 安全

- JWT Token有效期2小时，Refresh Token有效期7天
- 密码bcrypt加密，强度因子>=10
- 所有API接口鉴权，敏感操作二次验证
- SQL注入防护（MyBatis参数化查询）
- XSS防护（前端转义 + 后端过滤）
- CSRF防护（Token验证）
- 支付回调签名验证

### 6.3 兼容性

- PC：Chrome 80+、Firefox 80+、Safari 14+、Edge 80+
- 小程序：微信基础库 2.20+
- 公众号H5：iOS Safari 14+、Android Chrome 80+
