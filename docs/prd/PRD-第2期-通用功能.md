## 1. 概述

### 1.1 本期目标

补齐主流B2B2C商城通用功能，包括营销活动、积分商城、店铺可视化装修、三级分销体系，使商城具备完整的运营工具链。

### 1.2 核心价值

- 运营人员可独立开展营销活动，提升GMV
- 商户可自主装修店铺，提升品牌形象
- 分销体系降低获客成本
- 积分体系提升用户粘性

### 1.3 依赖前置条件

- 第一期MVP闭环已完成
- 商品/订单/支付等基础链路可用
- 多租户框架已就绪

## 4. 业务流程

### 4.1 秒杀流程

```
运营创建秒杀活动 → 设置时间段/商品/价格/库存/限购
→ 活动开始前：库存预热到Redis
→ 活动开始：C端展示倒计时+秒杀入口
→ 用户点击抢购 → Redis原子扣减库存
→ 下单（MQ异步）→ 支付 → 发货
→ 支付超时 → Redis回补库存
→ 活动结束：下架秒杀入口
```

### 4.2 拼团流程

```
商户创建拼团活动 → 设置成团人数/时间/价格
→ 用户开团（创建拼团记录）
→ 其他用户参团
→ 达到成团人数 → 拼团成功 → 发货
→ 超时未成团 → 拼团失败 → 自动退款
```

### 4.3 分销佣金计算

```
用户通过分销员B的推广链接下单
→ 订单完成（确认收货）→ 触发佣金计算
→ B获得一级佣金（订单金额 × 一级比例，如10%）
→ B的上级C获得二级佣金（订单金额 × 二级比例，如5%）
→ C的上级D获得三级佣金（订单金额 × 三级比例，如2%）
→ 佣金进入冻结状态（防退款后佣金已提现）
→ 7天无售后 → 佣金可提现
→ 分销员申请提现 → 平台审核 → 打款
```

### 4.4 可视化装修流程

```
商户进入装修编辑器
→ 从组件库选择组件（轮播图/商品列表/广告位...）
→ 拖拽到画布指定位置
→ 配置组件属性（如：轮播图上传图片/设置链接）
→ 点击预览 → 查看效果
→ 保存草稿（不发布）
→ 或直接发布 → C端首页即时更新
→ 可导出模板 → 备份/共享
→ 可导入模板 → 快速复用
```

## 5. 非功能需求

### 5.1 性能

| 指标 | 要求 |
|------|------|
| 秒杀接口 | 支持3000 QPS |
| 拼团并发 | 支持500人同时参团 |
| 装修编辑器 | 组件拖拽响应 < 100ms |
| 装修页面渲染 | 发布后C端加载 < 2s |

### 5.2 安全

- 秒杀接口限流（令牌桶）
- 分销佣金计算幂等（防止重复计算）
- 优惠券领取防刷（同一用户限领N张）
- 装修组件XSS防护（富文本过滤）

## 6. 验收标准

- [ ] 优惠券：创建→领取→购物车展示→下单使用→金额正确
- [ ] 秒杀：倒计时→抢购→库存扣减→超时回补
- [ ] 拼团：开团→参团→成团发货/失败退款
- [ ] 分销：申请→审核→推广→下单→佣金计算→提现
- [ ] 装修：拖拽→配置→预览→发布→C端更新
- [ ] 积分：获取→明细→兑换→有效期
