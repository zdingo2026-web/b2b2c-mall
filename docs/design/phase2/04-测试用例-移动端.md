## 一、文档概述

### 1.1 测试范围

本文档覆盖移动端二期新增的 20 个页面，按模块分组：

| 模块 | 页面数 | 页面列表 |
|------|:------:|---------|
| 首页增强 | 2 | 首页(增强)、商品分类 |
| 营销活动 | 4 | 秒杀专区页、拼团专区页、团购专区页、优惠活动页 |
| 积分商城 | 3 | 积分商城首页、积分商品详情、积分兑换确认 |
| 购物车/结算增强 | 2 | 购物车(增强)、结算页(增强) |
| 会员中心 | 7 | 我的优惠券、积分中心、签到页、红包列表、分销中心、分销推广海报、收藏合并(商品/店铺) |
| 店铺 | 1 | 店铺首页 |
| **合计** | **19+1** | 收藏合并含商品和店铺两个子页面 |

> 收藏合并(商品)和收藏合并(店铺)为同一页面路径的不同Tab/视图，分别列出测试用例。

### 1.2 截图+AI分析模式说明

每个测试用例包含**截图要求**和**AI分析判断标准**。执行测试时：

1. 按步骤操作，在指定步骤截取指定区域/状态的截图
2. 将截图交给AI分析，AI根据判断标准输出"通过/未通过/未确定"
3. 判断标准使用具体可验证的条件（如"列表中存在>=5个商品卡片"、"按钮文字为'立即抢购'且文字颜色为红色(#E11148)"、"2列网格布局，左列和右列各>=2个商品卡片"等）

### 1.3 模拟数据策略

- 所有列表页面必须先注入模拟数据（至少5条完整记录），确保页面有数据可展示
- **无数据的列表页面 = 测试未通过**
- 模拟数据以 JSON 格式给出，标注 API 调用方式
- 每条模拟数据必须包含页面所需展示的全部字段

### 1.4 双端测试策略

| 维度 | H5 | 微信小程序 |
|------|-----|-----------|
| 测试环境 | Chrome Mobile模拟器(375x812) / 真机浏览器 | 微信开发者工具 / 真机微信8.0+ |
| 登录方式 | 手机号+密码登录 | wx.login + 手机号绑定 |
| 支付方式 | H5支付(JSAPI/扫码) | 小程序支付(wx.requestPayment) |
| 分享方式 | 复制链接 | wx.shareAppMessage |
| 海报保存 | 长按保存 | wx.saveImageToPhotosAlbum |
| 二维码 | 普通二维码(带URL) | 小程序码 |
| 签到 | 页面内按钮 | 首次弹窗提醒+页面按钮 |
| 截图命名前缀 | TC-H- | TC-H-MP- |

### 1.5 通过标准

| 判定 | 条件 |
|------|------|
| 通过 | 截图和AI分析全部满足判断标准 |
| 未通过 | 任意判断标准不满足（含列表无数据） |
| 阻塞 | 前置条件不满足，无法执行测试步骤 |
| 跳过 | 当前环境不支持该测试（如H5环境测试小程序专属功能） |

### 1.6 优先级定义

| 优先级 | 定义 | 覆盖要求 |
|-------|------|---------|
| P0 | 核心功能，阻断性 | 100%执行 |
| P1 | 重要功能，影响体验 | 重点执行 |
| P2 | 辅助功能，体验优化 | 抽查执行 |
