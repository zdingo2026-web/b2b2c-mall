#### 1.1.1 优惠券体系

**1.1.1.1 优惠券模板（coupon_template）**

| minAmount | DECIMAL(10,2) | DEFAULT 0.00 | 使用门槛金额，0表示无门槛 |
| maxDiscount | DECIMAL(10,2) | DEFAULT NULL | 满折券折扣上限金额，满减/无门槛为NULL |

| issuerType | TINYINT | NOT NULL | 1-平台 2-商户 |
| issuerId | BIGINT | NOT NULL | 发放主体ID（tenantId=0为平台） |
| applyScope | TINYINT | DEFAULT 0 | 0-全店 1-指定分类 2-指定商品 |
| applyCategoryIds | VARCHAR(500) | DEFAULT NULL | 适用分类ID列表JSON（applyScope=1时） |

| validEndTime | DATETIME | DEFAULT NULL | 固定结束时间（validType=1时） |
| canStackWithSeckill | TINYINT | DEFAULT 0 | 秒杀商品是否可用：0-否 1-是 |
| activityId | BIGINT | DEFAULT NULL | 关联活动ID（用于新人礼包等自动发放场景） |
| status | TINYINT | DEFAULT 1 | 0-禁用 1-启用 |

**1.1.1.2 用户优惠券（member_coupon 重构）**

| 属性 | 类型 | 约束 | 说明 |
|------|------|------|------|
| couponTemplateId | BIGINT | NOT NULL | 关联优惠券模板ID |
| sourceType | TINYINT | DEFAULT 1 | 1-主动领取 2-系统发放(新人礼包) 3-活动赠送 |
| sourceId | BIGINT | DEFAULT NULL | 来源ID（活动ID/礼包ID） |
| orderId | BIGINT | DEFAULT NULL | 使用时关联的订单ID |
| validStartTime | DATETIME | NOT NULL | 生效时间（根据模板规则计算） |
| validEndTime | DATETIME | NOT NULL | 过期时间（根据模板规则计算） |

**1.1.1.3 优惠券金额计算规则**

```
满减券：discount = MIN(couponValue, totalAmount) 当 totalAmount >= minAmount
满折券：discount = totalAmount * (1 - couponValue/10)
        若 maxDiscount != NULL，则 discount = MIN(discount, maxDiscount)
满赠券：不算折扣金额，在订单项中添加赠品（赠品价格为0，标记为赠品）
无门槛减：discount = MIN(couponValue, totalAmount)
```

**边界条件：**
- 满减券：totalAmount < minAmount 时不可用
- 满折券：couponValue 必须 >= 0.1 且 < 10（即最多1折，最少99折）
- 满折券：discount 计算结果四舍五入到分，使用 BigDecimal.setScale(2, RoundingMode.HALF_UP)
- 优惠券优惠后，payAmount >= 0.01（最低支付1分钱）
- 赠品库存必须充足，否则领取时提示"赠品库存不足"

**1.1.1.4 优惠券领取规则**

- 每次领取先检查 limitPerUser，同一用户对同一模板已领数量 >= limitPerUser 时拒绝
- 领取时原子扣减 remainCount：先 Redis DECR，再 DB 更新（防超发）
- 若 Redis 返回 < 0，执行 Redis INCR 回补并返回"已领完"
- remainCount = 0 的模板前端展示"已领完"
- 优惠券状态流转：未使用(0) -> 已使用(1) / 已过期(2)

**1.1.1.5 优惠券使用规则**

- 下单时选择优惠券，系统校验：状态为"未使用"、有效期内、金额满足门槛、商品在适用范围内
- 每个子订单（按商户拆分）最多使用1张商户券 + 1张平台券
- 支付成功后，优惠券状态变为"已使用"，记录 orderId 和 useTime
- 订单取消/退款时，优惠券回退为"未使用"状态（仅在未过期的情况下；已过期则不回退）

**1.1.1.6 优惠券过期处理**

- 定时任务（每小时）扫描 status=0 AND validEndTime < NOW() 的记录，批量更新为 status=2
- 乐观锁处理：UPDATE member_coupon SET status=2 WHERE id=? AND status=0 AND validEndTime < NOW()

**1.1.2.3 秒杀并发处理方案**

```
阶段一：活动预热
  - 活动开始前5分钟，定时任务将 seckillStock 加载到 Redis
    Key: mall:seckill:stock:{seckillProductId}  Value: seckillStock
  - 加载商品信息到 Redis Hash
    Key: mall:seckill:product:{seckillProductId}
  - 生成秒杀路径令牌（防刷）
    Key: mall:seckill:path:{memberId}:{seckillProductId}  Value: UUID  TTL: 60s

阶段二：秒杀下单
  1. 前端携带路径令牌请求秒杀接口
  2. 验证令牌有效性（Redis GET + DEL 原子操作，一次性使用）
  3. 令牌桶限流（Redis + Lua脚本，每秒放行QPS上限）
  4. Redis Lua脚本原子操作：
     a. 判断用户是否已购买（Set: mall:seckill:bought:{seckillProductId}）
     b. 扣减 Redis 库存（DECR mall:seckill:stock:{seckillProductId}）
     c. 若库存 < 0，回补并返回失败
     d. 将用户ID加入已购买Set
  5. Redis扣减成功 -> 发MQ消息（seckill.order.queue）
  6. 消费者异步创建秒杀订单：再次校验DB库存、创建订单(orderType=2, 秒杀价)、扣减DB库存
  7. 用户轮询秒杀结果接口（mall:seckill:result:{memberId}:{seckillProductId}）

阶段三：支付与超时
  - 秒杀订单支付超时时间：5分钟（短于普通订单的30分钟）
  - 超时未支付：MQ消费回补 Redis 库存 + DB 库存 + 移除已购买Set

阶段四：活动结束
  - 定时任务将未售完的秒杀库存回补到商品SKU库存
  - 清理 Redis 缓存
```

**1.1.2.4 秒杀边界条件与异常处理**

| 场景 | 处理方式 |
|------|---------|
| 活动未开始 | 返回错误码，前端展示倒计时 |
| 活动已结束 | 返回错误码，前端展示"已结束" |
| 商品已下架 | Redis预热时检查，下架商品不预热 |
| 秒杀价 >= 原价 | 创建活动时校验拒绝 |
| 秒杀库存 > SKU库存 | 创建活动时校验拒绝 |
| 用户重复购买 | Redis Set 去重，DB 唯一索引兜底 |
| MQ消息丢失 | 消息持久化 + 手动ACK + 死信队列重试 |
| MQ重复消费 | 订单创建幂等（基于 seckillProductId + memberId 唯一） |
| Redis宕机 | 降级为DB行锁扣减，前端提示"系统繁忙" |
| 限流触发 | 返回 HTTP 429，前端展示"手慢了" |

**1.1.3.4 拼团流程详细规则**

```
开团：
  1. 用户选择商品 -> 点击"发起拼团"
  2. 系统校验：活动进行中 + 用户未参与同团 + 活动未达最大开团数
  3. 创建拼团记录（groupStatus=0, currentNum=1, expireTime=NOW()+duration）
  4. 创建拼团订单（orderType=3, 拼团价, 支付超时=10分钟）
  5. 用户支付后，等待其他人参团

参团：
  1. 用户选择已有拼团记录 -> 点击"参与拼团"
  2. 系统校验：拼团中 + 未满员 + 未过期 + 用户未参与同团
  3. 原子操作：currentNum + 1
  4. 若 currentNum == groupNum -> 拼团成功：更新groupStatus=1，所有订单推进为待发货
  5. 若 currentNum < groupNum -> 等待更多人

超时未成团：
  1. 定时任务（每分钟）扫描 groupStatus=0 AND expireTime < NOW() 的记录
  2. 更新 groupStatus=2, failTime=NOW()
  3. 批量取消相关订单，已支付的订单触发自动退款，释放库存
```

**1.1.3.5 拼团边界条件**

| 场景 | 处理方式 |
|------|---------|
| 团长未支付 | 团长订单超时取消，拼团记录自动失败，其他成员订单也取消退款 |
| 成员未支付 | 该成员订单超时取消，currentNum回减1 |
| 拼团中活动结束 | 活动结束时仍在拼团中的记录继续，直到成团或超时 |
| 同一用户重复参团 | 同一activityId下，用户只能参加一个未完成的团 |
| 退款后库存回补 | 拼团失败退款时，库存回补到SKU |

### 1.2 营销活动规则矩阵

#### 1.2.1 优惠叠加规则

| 序号 | 组合场景 | 是否可叠加 | 优先级 | 说明 |
|------|---------|-----------|--------|------|
| 1 | 平台满减券 + 商户满减券 | 可叠加 | 平台券后算 | 不同发放主体 |
| 2 | 平台满折券 + 商户满折券 | 可叠加 | 平台券后算 | 同上 |
| 3 | 平台满减券 + 商户满折券 | 可叠加 | 先折后减 | 先算商户折扣，再算平台满减 |
| 4 | 平台满折券 + 商户满减券 | 可叠加 | 先减后折 | 先算商户满减，再算平台满折 |
| 5 | 同主体两张满减券 | 互斥 | - | 同一发放主体只能用1张 |
| 6 | 同主体两张满折券 | 互斥 | - | 同上 |
| 7 | 同主体满减 + 满折 | 互斥 | - | 同一主体只选其一 |
| 8 | 优惠券 + 秒杀商品 | 可配置 | 优惠在后 | 取决于 canUseCoupon |
| 9 | 优惠券 + 拼团商品 | 互斥 | - | 拼团价已享优惠 |
| 10 | 优惠券 + 团购商品 | 互斥 | - | 团购价已享优惠 |
| 11 | 优惠券 + 限时折扣商品 | 可叠加 | 折扣先算 | 限时折扣价先算，再算优惠券 |
| 12 | 秒杀 + 拼团 | 互斥 | - | 同一商品不可同时参与 |
| 13 | 拼团 + 团购 | 互斥 | - | 同上 |
| 14 | 限时折扣 + 拼团 | 互斥 | - | 同上 |
| 15 | 限时折扣 + 团购 | 互斥 | - | 同上 |
| 16 | 首单优惠 + 优惠券 | 可叠加 | 首单优先 | 首单优惠先算，再算优惠券 |
| 17 | 红包 + 优惠券 | 可叠加 | 优惠优先 | 优惠券先算，红包最后抵扣 |
| 18 | 积分抵扣 + 优惠券 | 可叠加 | 优惠优先 | 优惠券先算，积分抵扣在后 |
| 19 | 多种活动商品混合下单 | 按商品拆分 | - | 不同活动类型商品不可混用同一张券 |

#### 1.2.2 叠加规则核心算法

```
1. 按商品计算活动价（秒杀价/拼团价/团购价/折扣价/原价）
2. 按商户分组，计算各组 totalAmount
3. 商户优惠券抵扣
4. 首单优惠抵扣
5. 汇总全订单 grandTotal
6. 平台优惠券抵扣
7. 积分抵扣（每单最多抵扣30%）
8. 红包抵扣
9. + 运费
10. payAmount = MAX(grandTotal, 0.01)
```

### 1.3 分销佣金计算公式和结算流程

#### 1.3.1 三级分销模型

```
D(三级上级) -> C(二级上级) -> B(直接推广员) -> A(消费者)
佣金基数 = payAmount - freightAmount（不含运费）
一级佣金 = 佣金基数 * oneLevelRate / 100
二级佣金 = 佣金基数 * twoLevelRate / 100
三级佣金 = 佣金基数 * threeLevelRate / 100
约束：三级之和 <= 30%；单笔最低0.01元
```

#### 1.3.2 佣金优先级

商品级佣金 > 分类级佣金 > 全局级佣金

#### 1.3.3 佣金结算流程

```
1. 订单确认收货 -> 佣金计算 -> 创建佣金记录(冻结态)
2. 冻结期7天到期 -> 可提现
3. 退款处理：冻结态直接删除、可提现态扣减余额、已提现态标记待追回
4. 提现申请 -> 平台审核 -> 打款
5. 最低提现金额：100元（sys_config配置）
```

### 2.4 数据流转设计

#### 2.4.1 营销活动对订单金额的影响流

```
商品原价 -> [活动价替换] -> [商户优惠券] -> [首单优惠] -> [汇总] -> [平台优惠券] -> [积分抵扣] -> [红包抵扣] -> [+运费] -> payAmount
```

#### 2.4.2 积分与订单关联

#### 2.4.3 分销佣金与订单关联

订单确认收货 -> 判断分销来源 -> 计算三级佣金 -> 冻结7天 -> 可提现

#### 2.5.1 优惠券状态机

```
[不存在] --领取--> [未使用(0)] --使用--> [已使用(1)] --取消(未过期)--> [未使用(0)]
                        |                       |
                      过期                    取消(已过期)
                        |                       |
                    [已过期(2)]              不回退
```

```
      |
    超时
      |
[拼团失败(2)] --> 所有订单取消/退款
```

```
[待审核(0)] --通过--> [已通过(1)] --冻结--> [已冻结(3)] --解冻--> [已通过(1)]
      |                                      |
    拒绝                                  封禁
      |                                      |
[已拒绝(2)]                              [已封禁(4)]
```

```
[冻结(0)] --到期--> [可提现(1)] --申请--> [提现中(2)] --通过--> [已提现(3)]

退款                审核拒绝            审核拒绝
  |                    |                   |
[已扣除(4)]         [可提现(1)]          [可提现(1)]

```

```
               |
          积分+现金--支付--> [已完成(1)] / --超时--> [已取消(2)]
```

**PromotionEngine** -- 营销引擎核心：

```java
@Service
@RequiredArgsConstructor
public class PromotionEngine {

    private final List<PromotionStrategy> strategies;
    private final CouponService couponService;

    /**
     * 计算最优优惠组合
     * 规则：
     * 1. 秒杀/拼团/团购为互斥活动，选中时独占优惠
     * 2. 平台券+商家券可叠加
     * 3. 同类型券互斥（满减+满折不可叠加）
     * 4. 活动型优惠与优惠券的叠加由后台配置决定
     */
    public PromotionBestResult calculateBest(PromotionContext context) {
        // 1. 如果是活动型订单(秒杀/拼团/团购)，直接走活动策略
        if (context.getOrderType() != null && context.getOrderType() > 1) {
            PromotionStrategy activityStrategy = getStrategy(context.getOrderType());
            if (activityStrategy != null && activityStrategy.isAvailable(context)) {
                PromotionResult activityResult = activityStrategy.calculate(context);
                // 检查活动是否允许叠加优惠券
                if (isCouponStackableWithActivity(context.getActivityId())) {
                    PromotionResult couponResult = calculateBestCoupon(context, activityResult);
                    return mergeResults(activityResult, couponResult);
                }
                return PromotionBestResult.single(activityResult);
            }
        }

        // 2. 普通订单：计算最优优惠券组合
        return calculateBestCouponCombination(context);
    }

    /**
     * 最优优惠券组合算法（贪心）
     * 步骤：
     * a. 从用户可用券中按tenantId分组
     * b. 平台券组选优惠力度最大的1张
     * c. 每个商家券组各选优惠力度最大的1张
     * d. 验证叠加规则（同类型互斥）
     * e. 返回总优惠最大的组合
     */
    private PromotionBestResult calculateBestCouponCombination(PromotionContext context) {
        // 获取用户所有可用优惠券
        List<MemberCoupon> availableCoupons = couponService.getAvailableCoupons(
            context.getMemberId(), context.getTenantId());

        // 按发放主体分组: 平台券 vs 商家券
        Map<Long, List<MemberCoupon>> groupedByIssuer = availableCoupons.stream()
            .collect(Collectors.groupingBy(MemberCoupon::getTenantId));

        // 平台券(tenantId=0)中选最优
        List<MemberCoupon> platformCoupons = groupedByIssuer.getOrDefault(0L, Collections.emptyList());
        MemberCoupon bestPlatformCoupon = selectBestCoupon(platformCoupons, context);

        // 商家券中每家选最优
        Map<Long, MemberCoupon> bestMerchantCoupons = new LinkedHashMap<>();
        for (Map.Entry<Long, List<MemberCoupon>> entry : groupedByIssuer.entrySet()) {
            if (entry.getKey() != 0L) {
                MemberCoupon best = selectBestCoupon(entry.getValue(), context);
                if (best != null) {
                    bestMerchantCoupons.put(entry.getKey(), best);
                }
            }
        }

        // 验证叠加规则：满减+满折互斥
        return validateAndBuildResult(bestPlatformCoupon, bestMerchantCoupons, context);
    }

    private MemberCoupon selectBestCoupon(List<MemberCoupon> coupons, PromotionContext context) {
        // 计算每张券的实际优惠金额，取最大
        return coupons.stream()
            .filter(c -> isCouponApplicable(c, context))
            .max(Comparator.comparing(c -> calculateCouponDiscount(c, context)))
            .orElse(null);
    }
}
```

#### 3.2.2 优惠券叠加引擎

叠加规则矩阵实现：

```java
@Component
@RequiredArgsConstructor
public class CouponStackValidator {

    /**
     * 叠加规则矩阵
     * key: (券A类型, 券B类型)
     * value: true=可叠加, false=互斥
     */
    private static final Map<String, Boolean> STACK_MATRIX = new HashMap<>();
    static {
        // 满减 + 满减 = 互斥
        STACK_MATRIX.put(key(1, 1), false);
        // 满减 + 满折 = 互斥
        STACK_MATRIX.put(key(1, 2), false);
        // 满折 + 满折 = 互斥
        STACK_MATRIX.put(key(2, 2), false);
        // 满减 + 满赠 = 可叠加（不同维度的优惠）
        STACK_MATRIX.put(key(1, 3), true);
        // 满折 + 满赠 = 可叠加
        STACK_MATRIX.put(key(2, 3), true);
        // 满赠 + 满赠 = 互斥
        STACK_MATRIX.put(key(3, 3), false);
    }

    private static String key(int typeA, int typeB) {
        return Math.min(typeA, typeB) + ":" + Math.max(typeA, typeB);
    }

    /**
     * 判断两张券是否可叠加使用
     * 规则：不同发放主体(平台vs商家)且类型兼容时可叠加
     */
    public boolean canStack(MemberCoupon couponA, MemberCoupon couponB) {
        // 同一张券不可叠加自身
        if (couponA.getId().equals(couponB.getId())) {
            return false;
        }
        // 同发放主体 + 同类型 = 必然互斥
        if (couponA.getTenantId().equals(couponB.getTenantId())) {
            return false;
        }
        // 不同发放主体：查叠加矩阵
        return STACK_MATRIX.getOrDefault(
            key(couponA.getCouponType(), couponB.getCouponType()), false);
    }
}
```

```java
@RequiredArgsConstructor
public class SeckillService {

    private final StringRedisTemplate redisTemplate;
    private final SeckillActivityMapper seckillActivityMapper;
    private final SeckillSkuMapper seckillSkuMapper;

    /**
     * 预热秒杀活动到Redis
     * 由定时任务在活动开始前5分钟调用

    public void preloadSeckillActivity(Long activityId) {
        SeckillActivity activity = seckillActivityMapper.selectById(activityId);
        if (activity == null) return;

        String key = RedisKeyConstant.SECKILL_ACTIVITY + activityId;
        Map<String, String> hash = new HashMap<>();
        hash.put("stock", String.valueOf(activity.getTotalStock()));
        hash.put("price", activity.getSeckillPrice().toPlainString());
        hash.put("originalPrice", activity.getOriginalPrice().toPlainString());
        hash.put("limitPerUser", String.valueOf(activity.getLimitPerUser()));
        hash.put("startTime", String.valueOf(activity.getStartTime()
            .atZone(ZoneId.systemDefault()).toEpochSecond()));
        hash.put("endTime", String.valueOf(activity.getEndTime()
            .atZone(ZoneId.systemDefault()).toEpochSecond()));
        hash.put("status", "1");
        hash.put("allowCoupon", String.valueOf(activity.getAllowCoupon()));

        redisTemplate.opsForHash().putAll(key, hash);

        // 建立SKU反查索引
        List<SeckillSku> skus = seckillSkuMapper.selectList(
            new LambdaQueryWrapper<SeckillSku>()
                .eq(SeckillSku::getActivityId, activityId));
        for (SeckillSku sku : skus) {
            redisTemplate.opsForValue().set(
                RedisKeyConstant.SECKILL_SKU + sku.getSkuId(),
                String.valueOf(activityId));
        }

        // 设置过期时间 = 活动结束时间 + 1小时
        long expireSeconds = Duration.between(LocalDateTime.now(),
            activity.getEndTime().plusHours(1)).getSeconds();
        redisTemplate.expire(key, expireSeconds, TimeUnit.SECONDS);
    }
```

```lua
-- seckill_deduct.lua
-- 秒杀库存原子扣减 + 限购判断
--
-- KEYS[1]: mall:seckill:activity:{activityId}    -- 活动Hash
-- KEYS[2]: mall:seckill:bought:{activityId}:{memberId}  -- 用户已购数
--
-- ARGV[1]: quantity    -- 购买数量
-- ARGV[2]: limitPerUser -- 每人限购
-- ARGV[3]: currentTime -- 当前时间戳(秒)
-- ARGV[4]: startTime   -- 活动开始时间戳
-- ARGV[5]: endTime     -- 活动结束时间戳

-- 1. 检查活动时间
local startTime = tonumber(ARGV[4])
local endTime = tonumber(ARGV[5])
local currentTime = tonumber(ARGV[3])
if currentTime < startTime then
    return -1  -- 活动未开始
end
if currentTime > endTime then
    return -2  -- 活动已结束
end

-- 2. 检查库存
local stock = tonumber(redis.call('HGET', KEYS[1], 'stock'))
if stock == nil or stock < tonumber(ARGV[1]) then
    return -3  -- 库存不足
end

-- 3. 检查限购
local bought = tonumber(redis.call('GET', KEYS[2]) or '0')
if bought + tonumber(ARGV[1]) > tonumber(ARGV[2]) then
    return -4  -- 超出限购
end

-- 4. 扣减库存
redis.call('HINCRBY', KEYS[1], 'stock', -tonumber(ARGV[1]))

-- 5. 记录用户已购数
redis.call('INCRBY', KEYS[2], tonumber(ARGV[1]))
local ttl = endTime - currentTime + 3600  -- 活动结束后1小时过期
redis.call('EXPIRE', KEYS[2], ttl)

return 1  -- 成功
```

```java
            dto.setActivityId(message.getActivityId());

            item.setQuantity(message.getQuantity());

                message.getMemberId(), dto, message.getSeckillPrice());
```

#### 3.3.4 令牌桶限流实现

基于Redis + Lua的令牌桶，复用现有 `RateLimitUtil` 扩展：

```java
@Component
@RequiredArgsConstructor
public class TokenBucketUtil {

    private final StringRedisTemplate redisTemplate;
    private final DefaultRedisScript<Long> tokenBucketScript;

    /**
     * 令牌桶限流
     * @param key 限流key
     * @param capacity 桶容量（最大突发量）
     * @param rate 令牌生成速率（个/秒）
     * @return true=获取令牌成功, false=被限流
     */
    public boolean tryAcquire(String key, int capacity, double rate) {
        long now = System.currentTimeMillis();
        Long result = redisTemplate.execute(tokenBucketScript,
            Collections.singletonList(key),
            String.valueOf(capacity),
            String.valueOf(rate),
            String.valueOf(now));
        return result != null && result == 1L;
    }
}
```

Lua脚本 -- 令牌桶：

```lua
-- token_bucket.lua
-- KEYS[1]: bucket key
-- ARGV[1]: capacity
-- ARGV[2]: rate (tokens per second)
-- ARGV[3]: current timestamp (ms)

local key = KEYS[1]
local capacity = tonumber(ARGV[1])
local rate = tonumber(ARGV[2])
local now = tonumber(ARGV[3])

local bucket = redis.call('HMGET', key, 'tokens', 'lastTime')
local tokens = tonumber(bucket[1])
local lastTime = tonumber(bucket[2])

if tokens == nil then
    tokens = capacity
    lastTime = now
end

-- 计算新增令牌
local elapsed = (now - lastTime) / 1000
local newTokens = elapsed * rate
tokens = math.min(capacity, tokens + newTokens)

if tokens >= 1 then
    tokens = tokens - 1
    redis.call('HMSET', key, 'tokens', tokens, 'lastTime', now)
    redis.call('EXPIRE', key, math.ceil(capacity / rate) + 10)
    return 1
else
    redis.call('HMSET', key, 'tokens', tokens, 'lastTime', now)
    return 0
end
```

#### 3.3.5 动态URL防提前抢购

方案：秒杀入口URL包含动态token，在活动开始时才生成并下发。

```java
/**
 * 获取秒杀动态URL
 * 活动未开始返回倒计时，活动进行中返回带token的抢购URL
 */
public SeckillUrlResult getSeckillUrl(Long activityId, Long memberId) {
    SeckillActivity activity = seckillActivityMapper.selectById(activityId);
    LocalDateTime now = LocalDateTime.now();

    if (now.isBefore(activity.getStartTime())) {
        // 未开始：返回倒计时
        return SeckillUrlResult.countdown(activity.getStartTime());
    }

    if (now.isAfter(activity.getEndTime())) {
        return SeckillUrlResult.ended();
    }

    // 进行中：生成动态token
    // token = MD5(activityId + memberId + salt + startTime)
    String raw = activityId + ":" + memberId + ":"
        + SECKILL_SALT + ":" + activity.getStartTime();
    String token = DigestUtils.md5Hex(raw);

    // 缓存token到Redis，有效期=活动剩余时间
    String tokenKey = RedisKeyConstant.SECKILL_TOKEN + activityId + ":" + memberId;
    redisTemplate.opsForValue().set(tokenKey, token,
        Duration.between(now, activity.getEndTime()));

    String url = "/api/v1/member/seckill/execute?activityId=" + activityId
        + "&token=" + token;
    return SeckillUrlResult.available(url);
}

/**
 * 抢购执行时校验token
 */
private void validateSeckillToken(Long activityId, Long memberId, String token) {
    String tokenKey = RedisKeyConstant.SECKILL_TOKEN + activityId + ":" + memberId;
    String cachedToken = redisTemplate.opsForValue().get(tokenKey);
    if (cachedToken == null || !cachedToken.equals(token)) {
        throw new BusinessException("非法抢购请求");
    }
    // 一次性token，用后即焚
    redisTemplate.delete(tokenKey);
}
```

### 3.4 拼团方案

#### 3.4.1 开团/参团并发控制

拼团的核心并发场景是多人同时参团（最后1-2个名额）。使用Redis分布式锁 + 乐观锁双重保障。

```java
@Service
@RequiredArgsConstructor
public class GroupBuyService {

    private final StringRedisTemplate redisTemplate;
    private final GroupBuyActivityMapper groupBuyActivityMapper;
    private final GroupBuyRecordMapper groupBuyRecordMapper;

    /**
     * 开团
     */
    @Transactional(rollbackFor = Exception.class)
    public GroupBuyRecordVO openGroup(Long memberId, Long activityId, Long skuId, Integer quantity) {
        GroupBuyActivity activity = groupBuyActivityMapper.selectById(activityId);
        if (activity == null || activity.getStatus() != 1) {
            throw new BusinessException("拼团活动不存在或已结束");
        }
        LocalDateTime now = LocalDateTime.now();
        if (now.isBefore(activity.getStartTime()) || now.isAfter(activity.getEndTime())) {
            throw new BusinessException("拼团活动不在有效期内");
        }

        // 创建拼团记录
        GroupBuyRecord record = new GroupBuyRecord();
        record.setActivityId(activityId);
        record.setMemberId(memberId);
        record.setSkuId(skuId);
        record.setQuantity(quantity);
        record.setCurrentCount(1);
        record.setTargetCount(activity.getGroupSize());
        record.setStatus(GroupBuyStatusEnum.PENDING.getCode());
        record.setExpireTime(now.plusHours(activity.getExpireHours()));
        record.setTenantId(activity.getTenantId());
        groupBuyRecordMapper.insert(record);

        return toVO(record);
    }

    /**
     * 参团
     * 使用Redis原子操作保证成团人数不超限
     */
    @Transactional(rollbackFor = Exception.class)
    public GroupBuyRecordVO joinGroup(Long memberId, Long recordId, Long skuId, Integer quantity) {
        // 1. Redis分布式锁防止并发参团
        String lockKey = RedisKeyConstant.GROUP_JOIN_LOCK + recordId;
        String lockValue = UUID.randomUUID().toString();
        Boolean locked = redisTemplate.opsForValue()
            .setIfAbsent(lockKey, lockValue, 10, TimeUnit.SECONDS);
        if (locked == null || !locked) {
            throw new BusinessException("系统繁忙，请重试");
        }

        try {
            GroupBuyRecord record = groupBuyRecordMapper.selectById(recordId);
            if (record == null) {
                throw new BusinessException("拼团记录不存在");
            }

            // 2. 状态检查
            if (record.getStatus() != GroupBuyStatusEnum.PENDING.getCode()) {
                throw new BusinessException("该拼团已结束");
            }
            if (LocalDateTime.now().isAfter(record.getExpireTime())) {
                throw new BusinessException("该拼团已超时");
            }
            if (record.getCurrentCount() >= record.getTargetCount()) {
                throw new BusinessException("该拼团已满员");
            }
            if (record.getMemberId().equals(memberId)) {
                throw new BusinessException("不能参加自己开的团");
            }

            // 3. 乐观锁更新：currentCount + 1, 如果达到targetCount则status=SUCCESS
            int newCount = record.getCurrentCount() + 1;
            int newStatus = newCount >= record.getTargetCount()
                ? GroupBuyStatusEnum.SUCCESS.getCode()
                : GroupBuyStatusEnum.PENDING.getCode();

            LambdaUpdateWrapper<GroupBuyRecord> wrapper = new LambdaUpdateWrapper<>();
            wrapper.eq(GroupBuyRecord::getId, recordId)
                .eq(GroupBuyRecord::getCurrentCount, record.getCurrentCount())  // 乐观锁
                .set(GroupBuyRecord::getCurrentCount, newCount)
                .set(GroupBuyRecord::getStatus, newStatus);

            int rows = groupBuyRecordMapper.update(null, wrapper);
            if (rows == 0) {
                throw new BusinessException("拼团人数已满，请参加其他团");
            }

            // 4. 如果成团，发送成团通知
            if (newStatus == GroupBuyStatusEnum.SUCCESS.getCode()) {
                // 成团成功 -> 触发订单确认（发货）
                publishGroupSuccessEvent(recordId);
            }

            return toVO(record);
        } finally {
            // 释放锁（Lua脚本，防止误删他人锁）
            releaseLock(lockKey, lockValue);
        }
    }
}
```

#### 3.4.2 成团判断逻辑

成团判断在参团时通过乐观锁完成（见上方 `joinGroup` 方法），核心逻辑：

1. `currentCount + 1 >= targetCount` 即为成团
2. 使用 `UPDATE ... WHERE current_count = #{oldCount}` 乐观锁，确保最后一名参团者能准确更新状态
3. 成团后触发 `GroupSuccessEvent`，所有参团用户的订单进入"待发货"状态

#### 3.4.3 超时未成团退款

**方案选择：延迟队列优先，定时任务兜底**

选择原因：
- 延迟队列（RabbitMQ DLX + TTL）可精确到秒级触发，及时性好
- 定时任务每5分钟扫描一次作为兜底，防止MQ消息丢失

```java
/**
 * 开团时发送延迟消息
 * TTL = 拼团活动设定的超时小时数
 */
public void sendGroupTimeoutMessage(Long recordId, int expireHours) {
    // 每个拼团的TTL不同，需要为每个recordId创建独立的延迟队列
    // 实际实现：使用消息级别的TTL + DLX
    // RabbitMQ 3.5.9+ 支持per-message TTL
    rabbitTemplate.convertAndSend(
        GROUP_DELAY_QUEUE, "", String.valueOf(recordId),
        message -> {
            message.getMessageProperties().setExpiration(
                String.valueOf(expireHours * 3600 * 1000));
            return message;
        });
}
```

```java
public void handleGroupTimeout(String recordIdStr) {
    Long recordId = Long.valueOf(recordIdStr);
    GroupBuyRecord record = groupBuyRecordMapper.selectById(recordId);
    if (record == null) return;

    // 仅处理仍在PENDING状态的拼团
    if (record.getStatus() != GroupBuyStatusEnum.PENDING.getCode()) {
        return;
    }

    // 标记拼团失败

    wrapper.eq(GroupBuyRecord::getId, recordId)
        .eq(GroupBuyRecord::getStatus, GroupBuyStatusEnum.PENDING.getCode())
        .set(GroupBuyRecord::getStatus, GroupBuyStatusEnum.FAILED.getCode());
    groupBuyRecordMapper.update(null, wrapper);

    // 触发退款流程
    refundGroupBuyOrders(recordId);
}
```

```java
        for (int i = 0; i < components.size(); i++) {
            JSONObject comp = components.getJSONObject(i);
            String type = comp.getString("type");

                // 其他dataSource: "hot"/"new"/"category" 可扩展
            }
        }

        return root.toJSONString();
```

#### 3.5.5 预览/发布流程

```java
@Service
@RequiredArgsConstructor
public class DecoPageService {

    private final DecoPageMapper decoPageMapper;
    private final StringRedisTemplate redisTemplate;

    /**
     * 发布装修页面
     */
    @Transactional(rollbackFor = Exception.class)
    public void publishPage(Long pageId) {
        DecoPage page = decoPageMapper.selectById(pageId);
        if (page == null) {
            throw new BusinessException(ResultCode.DATA_NOT_FOUND);
        }

        // 1. XSS过滤：遍历组件，对rich_text的content进行过滤
        String safeJson = sanitizeRichText(page.getComponentsJson());

        // 2. 更新页面状态
        LambdaUpdateWrapper<DecoPage> wrapper = new LambdaUpdateWrapper<>();
        wrapper.eq(DecoPage::getId, pageId)
            .set(DecoPage::getStatus, 1)          // 已发布
            .set(DecoPage::getPublishedJson, safeJson)  // 发布快照
            .set(DecoPage::getVersion, page.getVersion() + 1);
        decoPageMapper.update(null, wrapper);

        // 3. 清除C端Redis缓存
        String cacheKey = RedisKeyConstant.DECO_PAGE
            + page.getTenantId() + ":" + page.getPageType();
        redisTemplate.delete(cacheKey);
    }

    /**
     * XSS过滤：仅对rich_text组件的content字段进行HTML净化
     */
    private String sanitizeRichText(String componentsJson) {
        JSONObject root = JSON.parseObject(componentsJson);
        JSONArray components = root.getJSONArray("components");

        for (int i = 0; i < components.size(); i++) {
            JSONObject comp = components.getJSONObject(i);
            if ("rich_text".equals(comp.getString("type"))) {
                JSONObject props = comp.getJSONObject("props");
                String content = props.getString("content");
                if (content != null) {
                    // 复用一期已有的HtmlSanitizeUtil
                    props.put("content", HtmlSanitizeUtil.sanitize(content));
                }
            }
        }

        return root.toJSONString();
    }

    /**
     * 保存草稿
     */
    @Transactional(rollbackFor = Exception.class)
    public void saveDraft(Long pageId, String componentsJson) {
        // 校验组件type合法性
        JSONObject root = JSON.parseObject(componentsJson);
        JSONArray components = root.getJSONArray("components");
        for (int i = 0; i < components.size(); i++) {
            String type = components.getJSONObject(i).getString("type");
            if (!ComponentTypeRegistry.isValid(type)) {
                throw new BusinessException("不支持的组件类型: " + type);
            }
        }

        LambdaUpdateWrapper<DecoPage> wrapper = new LambdaUpdateWrapper<>();
        wrapper.eq(DecoPage::getId, pageId)
            .set(DecoPage::getComponentsJson, componentsJson)
            .set(DecoPage::getStatus, 0);  // 草稿
        decoPageMapper.update(null, wrapper);
    }
}
```

```java
/**
 * 分销关系绑定拦截器
 * 在MemberAuthController.login成功后、或首页访问时执行
 */
@Component
@RequiredArgsConstructor
public class DistributionBindInterceptor {

    private final StringRedisTemplate redisTemplate;
    private final DistributionRelationMapper distributionRelationMapper;

    /**
     * 绑定分销关系
     * 规则：首次访问的inviter有效，已绑定关系的用户不会被覆盖（保护上级利益）
     */
    public void bindRelation(Long memberId, Long inviterId) {
        if (memberId == null || inviterId == null || memberId.equals(inviterId)) {
            return;
        }

        // 检查是否已有上级
        DistributionRelation existing = distributionRelationMapper.selectOne(
            new LambdaQueryWrapper<DistributionRelation>()
                .eq(DistributionRelation::getMemberId, memberId));

        if (existing != null) {
            return;  // 已有上级，不覆盖
        }

        // 查找inviter的上级链，构建完整3级关系
        DistributionRelation inviterRelation = distributionRelationMapper.selectOne(
            new LambdaQueryWrapper<DistributionRelation>()
                .eq(DistributionRelation::getMemberId, inviterId));

        Long parentLevel1 = inviterId;                           // 一级上级(直接推荐人)
        Long parentLevel2 = inviterRelation != null
            ? inviterRelation.getParentLevel1() : null;          // 二级上级
        Long parentLevel3 = inviterRelation != null
            ? inviterRelation.getParentLevel2() : null;          // 三级上级

        DistributionRelation relation = new DistributionRelation();
        relation.setMemberId(memberId);
        relation.setParentLevel1(parentLevel1);
        relation.setParentLevel2(parentLevel2);
        relation.setParentLevel3(parentLevel3);
        relation.setTenantId(TenantContext.getTenantId());
        distributionRelationMapper.insert(relation);

        // 写入Redis缓存（加速佣金计算时的查询）
        String key = RedisKeyConstant.DISTRIBUTION_RELATION + memberId;
        redisTemplate.opsForHash().putAll(key, new HashMap<String, String>() {{
            put("level1", String.valueOf(parentLevel1));
            put("level2", parentLevel2 != null ? String.valueOf(parentLevel2) : "0");
            put("level3", parentLevel3 != null ? String.valueOf(parentLevel3) : "0");
        }});
        redisTemplate.expire(key, 30, TimeUnit.DAYS);
    }
}
```

#### 3.6.2 三级佣金计算引擎

**触发时机**：订单确认收货后，通过MQ异步触发。

```java
@Service
@RequiredArgsConstructor
public class CommissionService {

    private final DistributionRelationMapper distributionRelationMapper;
    private final DistributionConfigMapper distributionConfigMapper;
    private final DistributionProductMapper distributionProductMapper;
    private final CommissionRecordMapper commissionRecordMapper;
    private final StringRedisTemplate redisTemplate;

    /**
     * 计算订单佣金（三级）
     * 幂等设计：基于orderItemId做幂等，同一订单项只计算一次佣金
     */
    @Transactional(rollbackFor = Exception.class)
    public void calculateCommission(Long orderItemId, Long memberId,
                                     Long tenantId, BigDecimal orderAmount,
                                     Long spuId) {
        // 1. 幂等检查
        String idempotentKey = RedisKeyConstant.COMMISSION_IDEMPOTENT + orderItemId;
        Boolean set = redisTemplate.opsForValue()
            .setIfAbsent(idempotentKey, "1", 24, TimeUnit.HOURS);
        if (set == null || !set) {
            log.warn("[Commission] Duplicate calculation: orderItemId={}", orderItemId);
            return;
        }

        // 2. 查询分销关系
        DistributionRelation relation = distributionRelationMapper.selectOne(
            new LambdaQueryWrapper<DistributionRelation>()
                .eq(DistributionRelation::getMemberId, memberId));
        if (relation == null) {
            return;  // 非分销用户订单，无佣金
        }

        // 3. 查询分销配置
        DistributionConfig config = distributionConfigMapper.selectOne(
            new LambdaQueryWrapper<DistributionConfig>()
                .eq(DistributionConfig::getTenantId, tenantId));
        if (config == null || config.getEnabled() != 1) {
            return;  // 分销未开启
        }

        // 4. 查询商品分销配置（如果商品有单独设置佣金比例，优先使用）
        DistributionProduct distProduct = distributionProductMapper.selectOne(
            new LambdaQueryWrapper<DistributionProduct>()
                .eq(DistributionProduct::getSpuId, spuId)
                .eq(DistributionProduct::getTenantId, tenantId));

        BigDecimal rate1 = distProduct != null && distProduct.getRateLevel1() != null
            ? distProduct.getRateLevel1() : config.getRateLevel1();
        BigDecimal rate2 = distProduct != null && distProduct.getRateLevel2() != null
            ? distProduct.getRateLevel2() : config.getRateLevel2();
        BigDecimal rate3 = distProduct != null && distProduct.getRateLevel3() != null
            ? distProduct.getRateLevel3() : config.getRateLevel3();

        // 5. 计算三级佣金
        createCommissionIfValid(relation.getParentLevel1(), orderItemId,
            orderAmount, rate1, 1, tenantId);
        createCommissionIfValid(relation.getParentLevel2(), orderItemId,
            orderAmount, rate2, 2, tenantId);
        createCommissionIfValid(relation.getParentLevel3(), orderItemId,
            orderAmount, rate3, 3, tenantId);
    }

    private void createCommissionIfValid(Long parentId, Long orderItemId,
                                          BigDecimal orderAmount, BigDecimal rate,
                                          int level, Long tenantId) {
        if (parentId == null || parentId == 0 || rate == null
            || rate.compareTo(BigDecimal.ZERO) <= 0) {
            return;
        }

        BigDecimal amount = orderAmount.multiply(rate)
            .setScale(2, RoundingMode.HALF_UP);

        CommissionRecord record = new CommissionRecord();
        record.setDistributorId(parentId);
        record.setOrderItemId(orderItemId);
        record.setCommissionAmount(amount);
        record.setCommissionRate(rate);
        record.setCommissionLevel(level);
        record.setStatus(CommissionStatusEnum.FROZEN.getCode());  // 冻结状态
        record.setUnfreezeTime(LocalDateTime.now().plusDays(7));   // 7天后解冻
        record.setTenantId(tenantId);
        commissionRecordMapper.insert(record);
    }
}
```

### 3.7 积分系统架构

#### 3.7.1 积分获取触发机制

| 触发场景 | 积分值计算 | 触发方式 |
|---------|----------|---------|
| 购物完成 | 实付金额 x 积分倍率 | 确认收货时MQ异步 |
| 每日签到 | 固定值 + 连续签到奖励 | API同步 |
| 商品评价 | 固定值 | 评价提交时MQ异步 |
| 首次完善资料 | 固定值(一次性) | API同步 |

```java
@Service
@RequiredArgsConstructor
public class PointsService {

    private final PointsDetailMapper pointsDetailMapper;
    private final PointsAccountMapper pointsAccountMapper;
    private final MemberMapper memberMapper;
    private final RabbitTemplate rabbitTemplate;

    /**
     * 增加积分（核心方法）
     * @param bizType 业务类型: 1-购物 2-签到 3-评价 4-系统发放
     * @param bizId   业务ID（订单ID/评价ID等，用于幂等）
     */
    @Transactional(rollbackFor = Exception.class)
    public void grantPoints(Long memberId, Integer bizType, String bizId,
                            Integer points, String remark) {
        // 1. 幂等检查：同一业务ID不重复发放
        Long count = pointsDetailMapper.selectCount(
            new LambdaQueryWrapper<PointsDetail>()
                .eq(PointsDetail::getBizType, bizType)
                .eq(PointsDetail::getBizId, bizId));
        if (count > 0) {
            log.warn("[Points] Duplicate grant: bizType={}, bizId={}", bizType, bizId);
            return;
        }

        // 2. 获取积分倍率（购物场景）
        int actualPoints = points;
        if (bizType == PointsBizTypeEnum.ORDER.getCode()) {
            PointsRule rule = getPointsRule(memberId);
            actualPoints = (int) (points * rule.getMultiplier());
        }

        // 3. 写入积分明细
        PointsDetail detail = new PointsDetail();
        detail.setMemberId(memberId);
        detail.setBizType(bizType);
        detail.setBizId(bizId);
        detail.setChangeAmount(actualPoints);
        detail.setChangeType(1);  // 1-增加 2-扣减
        detail.setRemark(remark);
        detail.setExpireTime(LocalDateTime.now().plusMonths(12));  // 默认12个月有效
        pointsDetailMapper.insert(detail);

        // 4. 更新积分账户（乐观锁）
        PointsAccount account = pointsAccountMapper.selectOne(
            new LambdaQueryWrapper<PointsAccount>()
                .eq(PointsAccount::getMemberId, memberId));
        if (account == null) {
            account = new PointsAccount();
            account.setMemberId(memberId);
            account.setBalance(actualPoints);
            account.setTotalEarned(actualPoints);
            pointsAccountMapper.insert(account);
        } else {
            int rows = pointsAccountMapper.addBalance(memberId, actualPoints);
            if (rows == 0) {
                throw new BusinessException("积分更新失败，请重试");
            }
        }

        // 5. 同步到Member.points字段（兼容一期代码）
        memberMapper.addPoints(memberId, actualPoints);
    }
}
```

#### 3.7.3 积分有效期处理

```java
/**
 * 定时任务：每天凌晨1点清理过期积分
 * 采用先进先出(FIFO)：最早获得的积分先过期
 */
@Scheduled(cron = "0 0 1 * * ?")
public void cleanExpiredPoints() {
    LocalDateTime now = LocalDateTime.now();

    // 查询所有过期的、未核销的积分明细(增加类型)
    List<PointsDetail> expired = pointsDetailMapper.selectList(
        new LambdaQueryWrapper<PointsDetail>()
            .eq(PointsDetail::getChangeType, 1)  // 增加类型
            .eq(PointsDetail::getExpired, 0)      // 未核销
            .le(PointsDetail::getExpireTime, now));

    // 按memberId分组
    Map<Long, List<PointsDetail>> grouped = expired.stream()
        .collect(Collectors.groupingBy(PointsDetail::getMemberId));

    for (Map.Entry<Long, List<PointsDetail>> entry : grouped.entrySet()) {
        Long memberId = entry.getKey();
        int totalExpired = entry.getValue().stream()
            .mapToInt(PointsDetail::getChangeAmount).sum();

        // 扣减积分账户
        pointsAccountMapper.deductBalanceIgnoreInsufficient(memberId, totalExpired);

        // 标记明细为已过期
        List<Long> detailIds = entry.getValue().stream()
            .map(PointsDetail::getId).collect(Collectors.toList());
        pointsDetailMapper.update(null,
            new LambdaUpdateWrapper<PointsDetail>()
                .in(PointsDetail::getId, detailIds)
                .set(PointsDetail::getExpired, 1));

        // 记录过期扣减明细
        PointsDetail expiryDetail = new PointsDetail();
        expiryDetail.setMemberId(memberId);
        expiryDetail.setBizType(PointsBizTypeEnum.EXPIRY.getCode());
        expiryDetail.setChangeAmount(totalExpired);
        expiryDetail.setChangeType(2);
        expiryDetail.setRemark("积分过期自动清理");
        pointsDetailMapper.insert(expiryDetail);

        // 同步Member.points
        memberMapper.deductPoints(memberId, totalExpired);
    }
}
```

#### 3.7.4 积分+现金混合兑换计算

```java
/**
 * 积分+现金混合兑换计算
 * 商品设置: pointsPrice(纯积分价), mixedPoints(混合所需积分), mixedCash(混合所需现金)
 */
public ExchangeCalcResult calculateExchange(Long pointsProductId, Long memberId) {
    PointsProduct product = pointsProductMapper.selectById(pointsProductId);
    PointsAccount account = pointsAccountMapper.selectOne(
        new LambdaQueryWrapper<PointsAccount>()
            .eq(PointsAccount::getMemberId, memberId));

    int userPoints = account != null ? account.getBalance() : 0;
    ExchangeCalcResult result = new ExchangeCalcResult();
    result.setPointsProductId(pointsProductId);

    if (product.getExchangeType() == 1) {
        // 纯积分兑换
        result.setExchangeType("points_only");
        result.setRequiredPoints(product.getPointsPrice());
        result.setRequiredCash(BigDecimal.ZERO);
        result.setCanAfford(userPoints >= product.getPointsPrice());
    } else if (product.getExchangeType() == 2) {
        // 积分+现金兑换
        result.setExchangeType("points_and_cash");
        result.setRequiredPoints(product.getMixedPoints());
        result.setRequiredCash(product.getMixedCash());
        result.setCanAfford(userPoints >= product.getMixedPoints());
    }

    // 如果积分不足，计算差额
    if (!result.isCanAfford()) {
        result.setPointsGap(result.getRequiredPoints() - userPoints);
    }

    return result;
}
```

以上是完整的二期"通用功能"技术方案设计。方案的核心设计决策总结如下：

1. **营销引擎采用策略模式**，与一期PaymentFactory的PayStrategy模式保持一致，降低团队认知成本。`PromotionEngine`统一调度，优惠券叠加采用贪心算法+规则矩阵。

2. **秒杀方案选择Redis Hash + Lua原子扣减**，而非String，因为一次HGETALL获取全部活动信息减少网络往返。动态URL token机制防止提前抢购，一次性token用后即焚。

3. **拼团超时采用延迟队列+定时任务双重保障**，延迟队列精确触发，定时任务兜底防止MQ消息丢失。参团并发用Redis分布式锁+数据库乐观锁双层保护。

4. **装修引擎采用JSON存储组件列表**，后端只校验type合法性和XSS过滤，props具体结构由前端组件库定义，实现前后端解耦。

5. **分销佣金计算采用MQ异步+幂等双重保障**，基于orderItemId做Redis SETNX幂等，数据库层面有查询兜底。7天冻结期防止退款后佣金被提现。

6. **积分系统采用PointsAccount精确账本+Member.points冗余双写**，保证与一期代码兼容。过期积分采用FIFO策略，定时任务凌晨清理。

7. **所有MQ消费者统一幂等模式**：关键路径用Redis SETNX+数据库状态检查双重保障，非关键路径用数据库查询即可。死信队列兜底消费失败的消息。

与现有代码的关键集成点：`OrderService.createOrder()`扩展营销计算分支、`OrderTimeoutConsumer`扩展秒杀库存回补、`PaySuccessConsumer`扩展积分/佣金触发、`MemberCoupon`扩展字段、`OrderMain.orderType`扩展枚举值。
