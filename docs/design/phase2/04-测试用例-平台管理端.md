## 一、文档概述

### 1.1 测试范围

本文档覆盖平台管理端二期新增的 25 个页面，按模块分组：

| 模块 | 页面数 | 页面列表 |
|------|:------:|---------|
| 营销管理 | 7 | 优惠券模板列表、优惠券模板详情、秒杀活动详情、秒杀商品配置、新人礼包设置(未启用)、新人礼包设置(已启用)、首单优惠设置 |
| 积分管理 | 4 | 积分规则设置、积分消费规则、积分商品列表、积分统计 |
| 分销管理 | 4 | 分销设置、佣金提现审核、分销订单列表、分销统计 |
| 商家完善 | 6 | 商家等级设置、商家评分查看、分类佣金设置、佣金结算列表、商家冻结管理、入驻开关设置 |
| 会员增强 | 4 | 红包批次管理、会员等级设置、支付密码重置、实名认证审核 |

### 1.2 截图+AI分析模式说明

每个测试用例包含**截图要求**和**AI分析判断标准**。执行测试时：

1. 按步骤操作，在指定步骤截取指定区域/状态的截图
2. 将截图交给AI分析，AI根据判断标准输出"通过/未通过/未确定"
3. 判断标准使用具体可验证的条件（如"列表中存在>=5行数据行"、"文本颜色为红色"、"Tag显示蓝色背景"等）

### 1.3 模拟数据策略

- 所有列表页面必须先注入模拟数据（至少5条完整记录），确保页面有数据可展示
- 无数据的列表页面 = 测试未通过
- 模拟数据以 JSON 格式给出，标注 API 调用方式
- 每条模拟数据必须包含页面所需展示的全部字段

### 1.4 通过标准

| 判定 | 条件 |
|------|------|
| 通过 | 截图和AI分析全部满足判断标准 |
| 未通过 | 任意判断标准不满足（含列表无数据） |
| 未确定 | AI无法从截图判断，需人工确认 |

### 1.5 优先级定义

| 优先级 | 定义 | 覆盖要求 |
|-------|------|---------|
| P0 | 核心功能，阻断性 | 100%执行 |
| P1 | 重要功能，影响体验 | 重点执行 |
| P2 | 辅助功能，体验优化 | 抽查执行 |
