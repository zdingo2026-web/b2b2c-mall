## 二、平台管理端页面设计

### 2.1 优惠券模板列表

**路由：** `/platform/promotion/coupon-list`

**页面功能：**
- 查看全平台优惠券模板列表（含平台券和商家券）
- 按条件筛选优惠券模板
- 新增/编辑/启用/禁用/删除优惠券模板
- 查看领取统计

**页面布局：**

```
+----------------------------------------------------------+
| 面包屑: 营销管理 > 优惠券模板列表                            |
+----------------------------------------------------------+
| 筛选区                                                     |
| [券名称] [券类型 v] [发放主体 v] [状态 v] [有效期 v]         |
| [搜索] [重置]                         [+ 新增优惠券]        |
+----------------------------------------------------------+
| 表格区                                                     |
| ID | 券名称 | 券类型 | 面值/折扣 | 门槛 | 发放主体 |        |
| 总量 | 剩余 | 限领 | 有效期 | 状态 | 操作                    |
+----------------------------------------------------------+
| 分页: 共X条 < 1 2 3 ... >                                 |
+----------------------------------------------------------+
```

**列表字段：**

| 字段 | 列宽 | 排序 | 筛选 | 说明 |
|------|------|------|------|------|
| ID | 80px | 支持 | - | 模板ID |
| 券名称 | min-width 150px | - | 输入框模糊搜索 | |
| 券类型 | 100px | - | 下拉：满减/满折/满赠/无门槛 | Tag颜色区分：满减蓝、满折绿、满赠橙、无门槛紫 |
| 面值/折扣 | 120px | - | - | 满减显示"减X元"，满折显示"X折"，满赠显示"赠品"，无门槛显示"减X元" |
| 使用门槛 | 100px | - | - | "满X元"或"无门槛" |
| 发放主体 | 100px | - | 下拉：平台/商家 | 平台显示蓝色Tag，商家显示灰色Tag + 商家名称 |
| 发放总量 | 80px | 支持 | - | |
| 剩余数量 | 80px | - | - | 剩余=0时红色加粗 |
| 每人限领 | 80px | - | - | |
| 有效期 | 180px | - | 下拉：未开始/进行中/已过期 | 固定时间段显示日期范围，领取后N天显示"领取后X天有效" |
| 状态 | 80px | - | 下拉：启用/禁用/已结束 | StatusTag组件 |
| 操作 | 200px | - | - | 查看/编辑/启用(禁用)/删除 |

**按钮操作：**

| 按钮 | 触发条件 | 操作结果 |
|------|---------|---------|
| 新增优惠券 | 始终可见 | 跳转到优惠券模板详情页(新增模式) |
| 搜索 | 点击 | 按筛选条件重新查询列表 |
| 重置 | 点击 | 清空所有筛选条件 |
| 查看 | 每行 | 打开详情抽屉(只读) |
| 编辑 | 每行(status!=已结束) | 跳转到优惠券模板详情页(编辑模式) |
| 启用/禁用 | 每行 | 确认弹窗 -> 切换status字段 |
| 删除 | 每行(remainCount=totalCount) | 二次确认弹窗 -> 逻辑删除 |

**批量操作：**
- 批量启用：选中多条status=0的记录 -> 确认弹窗 -> 批量更新
- 批量禁用：选中多条status=1的记录 -> 确认弹窗 -> 批量更新

## 五、关键交互流程图

### 5.1 秒杀抢购完整交互流

```
用户进入秒杀专区页
  |
  v
选择时间段Tab -> 加载该时段秒杀商品列表
  |
  v
查看倒计时:
  - 未开始: 灰色"即将开始"，可设"提醒我"
  - 进行中: 红色"立即抢购"
  - 已结束: 灰色"已结束"
  |
  v (进行中)
点击"立即抢购" -> 按钮loading
  |
  v
请求秒杀路径令牌: POST /api/v1/member/seckill/path
  |
  +-- 未登录 -> 跳转登录页
  +-- 未开始 -> Toast"活动尚未开始"
  +-- 已结束 -> Toast"活动已结束"
  +-- 成功 -> 获取令牌token
  |
  v
携带令牌请求秒杀: POST /api/v1/member/seckill/execute
  |
  +-- 403 令牌无效 -> Toast"请求无效"
  +-- 429 限流 -> Toast"手慢了"
  +-- 排队中 -> 返回requestId
  |
  v
前端轮询秒杀结果: GET /api/v1/member/seckill/result
间隔: 500ms递增，最多10次(5秒)
  |
  +-- null(排队中) -> 继续轮询
  +-- orderNo(成功) -> 跳转订单确认页
  +-- fail(soldout) -> Toast"商品已售罄"
  +-- fail(limit) -> Toast"已达限购数量"
  +-- 超时 -> Toast"系统繁忙"
  |
  v (成功)
跳转订单确认页(5分钟支付超时) -> 支付 -> 成功
  |
  v (支付超时)
订单取消 -> Redis回补库存 -> 推送"订单超时取消"
```

## 六、跨端一致性要求

### 6.1 同一功能在不同端的差异点

| 功能 | 移动端(H5) | 移动端(小程序) |
|------|-----------|--------------|
| 秒杀抢购 | 直接跳转确认页 | 直接跳转确认页 |
| 拼团分享 | 复制链接 | wx.shareAppMessage |
| 推广海报 | 长按保存 | wx.saveImageToPhotosAlbum |
| 推广二维码 | 普通二维码 | 小程序码 |
| 签到 | 页面内按钮 | 首次弹窗提醒+页面按钮 |
| 支付 | H5支付 | 小程序支付 |
| 登录 | 手机号+密码 | wx.login+手机号绑定 |
| 收藏 | 列表+左滑操作 | 列表+左滑操作 |
| 商品卡片 | 2列网格 | 2列网格 |
| 倒计时 | 数字块20px(40rpx) | 数字块40rpx |
| 装修渲染 | 全宽自适应 | 全宽自适应(rpx) |

### 6.2 必须保持一致的数据展示

| 数据项 | 一致性要求 | 说明 |
|--------|-----------|------|
| 商品价格 | 全端一致 | 活动价、原价、优惠金额 |
| 优惠券面值 | 全端一致 | 面值、门槛、有效期 |
| 积分余额 | 全端一致 | 实时同步，Redis缓存 |
| 佣金金额 | 全端一致 | 可提现余额、冻结金额 |
| 订单状态 | 全端一致 | 状态描述、操作按钮可见性 |
| 倒计时 | 全端同步 | 基于服务端时间校准 |
| 库存数量 | 全端一致 | 秒杀库存、积分商品库存 |
| 评分数据 | 全端一致 | 商家评分、商品评分 |

### 6.3 各端的特色交互

**管理后台（平台+商户）特色：**
- Element Plus表格+表单组件，信息密度高
- 装修编辑器拖拽操作，键盘快捷键(Ctrl+Z撤销, Ctrl+S保存)
- 弹窗/抽屉交互模式

**移动端H5特色：**
- 2列商品网格，触屏友好
- 底部Sheet弹出选择器(优惠券/地址/规格)
- 左滑操作(删除/加购)
- 下拉刷新、上拉加载
- 浏览器前进后退手势

**移动端小程序特色：**
- 微信授权快捷登录
- 小程序内分享(好友/群聊)
- 小程序码推广(扫码直达)
- 微信支付(免密码小额)
- 订阅消息(拼团结果/佣金到账)
- 海报保存到相册
- 客服消息(商品咨询)
