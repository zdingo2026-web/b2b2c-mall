## 一、文档概述

### 1.1 测试范围

本文档覆盖商户管理端二期新增的 16 个页面，按模块分组：

| 模块 | 页面数 | 页面列表 |
|------|:------:|---------|
| 营销管理 | 8 | 优惠券模板列表、优惠券模板详情、拼团活动列表、拼团活动详情、团购活动列表、团购活动详情、限时折扣列表、限时折扣详情 |
| 装修管理 | 4 | 装修编辑器、图片库、相册管理、模板管理 |
| 分销管理 | 3 | 分销商品管理、分销订单列表、分销统计 |
| 商家完善 | 1 | 数据看板 |

### 1.2 截图+AI分析模式说明

每个测试用例包含**截图要求**和**AI分析判断标准**。执行测试时：

1. 按步骤操作，在指定步骤截取指定区域/状态的截图
2. 将截图交给AI分析，AI根据判断标准输出"通过/未通过/未确定"
3. 判断标准使用具体可验证的条件（如"列表中存在>=5行数据行"、"文本颜色为红色(#E11148)"、"Tag显示蓝色背景"等）

### 1.3 模拟数据策略

- 所有列表页面必须先注入模拟数据（至少5条完整记录），确保页面有数据可展示
- **无数据的列表页面 = 测试未通过**
- 模拟数据以 JSON 格式给出，标注 API 调用方式
- 每条模拟数据必须包含页面所需展示的全部字段
- 模拟数据中时间范围需包含测试执行时间（进行中的活动）

### 1.4 通过标准

| 判定 | 条件 |
|------|------|
| 通过 | 截图和AI分析全部满足判断标准 |
| 未通过 | 任意判断标准不满足（含列表无数据） |
| 未确定 | AI无法从截图判断，需人工确认 |

### 1.5 优先级定义

| 优先级 | 定义 | 覆盖要求 |
|-------|------|---------|
| P0 | 核心功能，阻断性 | 100%执行 |
| P1 | 重要功能，影响体验 | 重点执行 |
| P2 | 辅助功能，体验优化 | 抽查执行 |

### 1.6 编号规则

| 前缀 | 端 | 示例 |
|------|-----|------|
| TC-M- | 商户管理端 | TC-M-COUPON-01 |

模块缩写：

| 缩写 | 模块 |
|------|------|
| COUPON | 优惠券 |
| GROUP | 拼团 |
| GROUPON | 团购 |
| DISCOUNT | 限时折扣 |
| DECORATION | 装修 |
| DISTRIB | 分销 |
| DASHBOARD | 数据看板 |

### 1.7 多租户隔离验证

商户管理端所有页面均需验证多租户隔离：商户A登录后仅可见本商户数据，不可见商户B数据。此验证在每个列表页面的基础数据展示用例中作为判断标准之一。
