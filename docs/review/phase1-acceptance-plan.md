## 三、Agent团队分工

### 3.1 团队结构

```
Phase1-Acceptance-Team
├── requirements-analyst    # 需求分析师 - PRD对照验收
├── backend-tester          # 后端测试 - 编译验证+单元测试+API冒烟
├── frontend-tester         # 前端测试 - 编译验证+页面冒烟
├── security-fixer          # 安全修复 - 修复CRITICAL/HIGH安全问题
└── devops                  # 部署 - Docker构建+部署+提供访问链接
```

### 3.2 Agent使用Skills要求

所有Agent必须使用对应agent-skills进行工作：

| Agent | 必用Skills | 用途 |
|-------|-----------|------|
| requirements-analyst | planner + code-reviewer | PRD对照验收、验收标准输出 |
| backend-tester | build-error-resolver + tdd-guide | 编译修复、测试运行 |
| frontend-tester | build-error-resolver + dart-build-resolver | 前端编译修复 |
| security-fixer | security-reviewer + java-reviewer | 安全问题修复 |
| devops | performance-optimizer | Docker部署、性能调优 |

### 3.3 并行工作流

```
Phase 1: 并行验证 (3 agents)
  ├── requirements-analyst: PRD对照检查 → 输出验收差距报告
  ├── backend-tester: 后端编译+测试 → 修复错误
  └── frontend-tester: 前端编译+构建 → 修复错误

Phase 2: 安全修复 (1 agent, 可与Phase 1并行)
  └── security-fixer: 修复C-01~C-03, H-01~H-05

Phase 3: 部署 (1 agent, Phase 1+2完成后)
  └── devops: Docker构建 → 部署 → 提供访问链接
```
